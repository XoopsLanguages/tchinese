說明：模板

描述

XOOPS 允許網站的實際呈現（“外觀”或使用者介面）與後端運行的程式碼之間進行某種程度的分離。前者由一個基於圖形佈景主題的複雜機制處理。一個佈景主題在其基本形式中，可能包含定義網站視覺介面的基本 HTML 模板，以及補充該視覺介面所需的樣式表和圖片。佈景主題可以在管理介面中切換，因此更改 XOOPS 網站的整體外觀只需更改一個選擇器的值並點擊“接受”即可。XOOPS 還為網站管理員提供了讓使用者在一些可用的佈景主題中進行選擇的功能。

一般來說，一個 佈景主題 是網站的高層次視覺框架：例如，佈景主題定義了網站的整體佈局；另一方面，它通常不定義區塊內內容的具體分佈。這由一個稱為 模板集 的不同實體處理。一個模板集是一組 HTML 模板，它們定義了構成 XOOPS 網站的所有區塊中內容的呈現方式。如果我們想用一個更清晰的比喻來說明，我們可以說佈景主題就像房子的建築師，而模板集就像室內裝潢師。前者設定牆壁並定義一般佈局，但後者負責“裝飾”這些牆壁。

如果活動佈景主題是一個簡單的佈景主題，只包含一個佈景主題檔案、CSS 檔案和一些圖片，XOOPS 將使用預設模板集來定義區塊的呈現方式。然而，一些複雜的佈景主題非常精緻，可以包含它們自己的模板集。在這種情況下，網站管理員必須使用管理介面來設定佈景主題和模板集。

佈景主題的魔法是如何實現的？也就是說，HTML 模板如何包含動態內容？這不是透過鏡子和煙霧來完成的，而是透過一個稱為 Smarty 的巧妙模板引擎來完成的，它實現了過去稱為 變數插值 的功能。簡單來說，我們可以說 XOOPS 系統使用 PHP 檔案來保存模組的邏輯（存取資料庫、分配變數、管理權限...），使用插入了 Smarty 變數的 HTML 檔案來管理實際的視覺介面，並使用 Smarty 引擎在構建最終使用者看到的頁面的 HTML 中的特定位置插入值和字串。總而言之，這對您來說意味著什麼可以很容易地總結出來：由於其佈景主題系統，XOOPS 讓網站設計師可以構建更具吸引力和視覺效果更佳的網站，而無需過多擔心底層發生了什麼。

複製和下載模板集

如果您想建立一個自訂模板集，您可以透過選擇“複製”連結來複製預設集（或任何其他模板集）（系統將提示您為新模板集命名）。與預設模板集不同，其他模板集可以自由編輯。您也可以透過選擇“下載”連結將模板集副本保存到磁碟，或使用“刪除”連結將其移除。您有時還可能會看到在尚未建立模板的新模組旁邊有一個額外的“產生”連結。

編輯模板

您可以透過瀏覽目錄樹狀結構來查看可用模板的完整列表。當您選擇一個模板時，XOOPS 將在一個編輯器中打開它，您可以在其中編輯它。

在編輯模板時，您必須啟用一個偏好設定，該偏好設定會在目前佈景主題的 `themes/your theme/templates` 目錄下有較新的檔案時，自動更新模組模板的 `.tpl` 檔案（系統模組 -> 偏好設定 -> 一般設定 -> 從 themes/your theme/templates 目錄更新模組模板 .tpl 檔案？ -> “是”）。一旦網站公開，就應該關閉此設定。

如果您不小心弄亂了一個模板，別忘了您可以從預設模板集中檢索一個“良好”的副本。如果您正在使用非標準模板，最好在開始修改之前將其副本保存到磁碟。
