說明：模組

描述

模組是利用 XOOPS 提供的基礎架構，讓網站管理員能夠為網站使用者提供動態或靜態內容的軟體結構，這些內容通常通過一個非常易於使用的 Web 介面生成。

XOOPS 的強大之處部分在於模組的管理方式。網站管理員可以輕鬆地安裝、卸載、啟用或停用任何特定的模組（除了系統模組，原因顯而易見）。您也可以重新命名模組，並決定它們在網站選單中向最終使用者顯示的順序（如果您希望它們顯示出來），因為您還可以提供隱藏功能，定義一些活躍但對一般使用者不可見的模組。例如，您可以通過在選單欄中的勾選圖標停用，將某個模組設為不在選單中顯示。以新聞模組為例，您可以隨後向選定的使用者群組發送私人訊息，告訴他們通過在 URL 欄中輸入地址來訪問該目錄，例如 http://www.yoursite.com/modules/news。

要更改模組的順序（這將反映在選單中），您只需將模組拖放到所需的位置即可。

模組安裝需要一些規劃，無論是針對要顯示的模組還是頁面上的佈局。建議查看其他 XOOPS 網站，了解哪些模組被用來顯示各種類型的內容。這可以通過打開任何 XOOPS 網站並將滑鼠懸停在左右側面板的選單連結上輕鬆完成。同時查看瀏覽器狀態欄中顯示的 URL。您應該會看到類似這樣的地址：http://the_site.com/modules/news/...。所有模組都儲存在 /modules/ 目錄中，因此地址的下一部分（在此範例中為 'news'）顯示了將展示該選單連結內容的模組。

如果您打開該連結的頁面，它將展示該模組的使用方式以及它顯示的內容類型。像這樣查看其他網站也將為您提供有關佈局和設計能力的一些很好的見解。
